# 用語統一ルール（CLAUDE.mdのサンプル）

社内やコミュニティで表記がばらつきやすい用語を統一するためのルール設定例です。

---

```markdown
# 用語統一ルール

## 表記ルール

以下の用語は、必ず右側の表記を使ってください。

| NG（使わない） | OK（正しい表記） |
|---------------|----------------|
| AI木曜会 → 木曜会 | AI木曜会（常に正式名称） |
| ユーザ | ユーザー（長音あり） |
| サーバ | サーバー（長音あり） |
| Web | Web（頭文字は大文字） |
| Eメール | メール |
| お問い合わせ | お問い合わせ（ひらがな） |
| 問合せ / 問い合せ | お問い合わせ |
| 下さい | ください（ひらがな） |
| 致します | いたします（ひらがな） |
| 頂く | いただく（ひらがな） |

## 固有名詞

| 表記 | 注意点 |
|------|--------|
| Claude | 「クロード」とカタカナ表記しない |
| Claude Code | 「ClaudeCode」とスペースなしにしない |
| VS Code | 「VSCode」「vscode」としない |
| GitHub | 「Github」「github」としない |
| Google スプレッドシート | 「スプシ」は社外文書では使わない |

## 数字の表記

- 1桁の数字はアラビア数字：1、2、3
- 日付：2026年2月15日（曜日）の形式
- 時刻：14:00〜15:00（24時間表記）
- 金額：$20、¥2,000（通貨記号＋カンマ区切り）
```

---

## 使いどころ

- チームで文書を作るとき（表記ゆれ防止）
- 外部向けの告知文・記事を書くとき
- ブランドガイドラインがある組織での利用

自分のチームでよく使う固有名詞や間違いやすい表記を追加してカスタマイズしてください。
