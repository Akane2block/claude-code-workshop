# ですます調ルール（CLAUDE.mdのサンプル）

「ですます調」と「である調」を使い分けたい場合のルール設定例です。

---

```markdown
# 文体ルール

## 基本方針
すべての出力は「です・ます」調で統一してください。

## 具体的なルール
- 文末は「〜です」「〜ます」「〜でした」「〜ました」で終えること
- 「〜だ」「〜である」は使わないこと
- 箇条書きの中でも「です・ます」調を維持すること
- ただし、見出し（# で始まる行）は体言止めでOK

## NG例とOK例

| NG | OK |
|----|-----|
| 参加者が増えた | 参加者が増えました |
| 重要なポイントである | 重要なポイントです |
| 以下の通りだ | 以下の通りです |
| 注意が必要 | 注意が必要です |
```

---

## 使いどころ

- 社外向けの文書を作成するとき
- 報告書やレポートを作成するとき
- 顧客向けメールの下書きを作るとき

逆に社内のカジュアルなSlack投稿なら「である調」や「だ・である調」の方が自然な場合もあります。
用途に合わせてルールを変えてみてください。
